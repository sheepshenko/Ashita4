Summary

FindAll is a plugin that provides inventory cacheing. In addition to the obvious function of locating items on various characters, FindAll can cache your local character's inventory so that it loads instantly upon zoning. Both of these functions are disabled until you've allowed the client to fully load inventory once. If you change to another character on the same account, you will once again have to wait until inventory fully loads for your own safety. Note that if you have FindAll on autoload, the initial zonein counts as long as you allow inventory to fully load before zoning again. You can search before these conditions are met, but new items will not be cached on a fresh login until inventory has fully loaded once. Note that in addition to the typed commands, you may double click a search's tab in ImGui mode to close only that search.


Commands

These commands should be typed directly into the FFXI console. They can all be prefixed with /fa or /findall.


Syntax: /fa config instantload [Optional: on/off/disabled/enabled]
Arguments:
on/off/enabled/disabled: Determines the state to put the InstantLoad setting in.
Effects: Allows the user to toggle whether instant inventory loading will be used. If a state is not specified, will alternate.
Example Syntax: /fa config instantload disabled


Syntax: /fa config cachetodisc [Optional: on/off/disabled/enabled]
Arguments:
on/off/enabled/disabled: Determines the state to put the CacheToDisc setting in.
Effects: Allows the user to toggle whether the current character's inventory will be backed up for searching. You can still search other characters on a character that isn't making backups. If a state is not specified, will alternate.
Example Syntax: /fa config cachetodisc enabled


Syntax: /fa config displaymax [Required: maximum]
Arguments:
maximum: New amount of search results to limit. Must be between 1 and 20.
Effects: Allows the user to change how many search results will display in ImGui mode.
Example Syntax: /fa config displaymax 8


Syntax: /fa config displaymode [Optional: imgui/chatlog]
Arguments:
imgui/chatlog: Determines the state to put the DisplayMode setting in.
Effects: Changes the display mode. If a mode is not specified, will alternate.
Example Syntax: /fa config displaymode imgui


Syntax: /fa config keyitemprefix [Optional: on/off/disabled/enabled]
Arguments:
on/off/enabled/disabled: Determines the state to put the KeyItemPrefix setting in.
Effects: When enabled, the user must type 'KI:' at the start of search terms to include key items. When disabled, all searches include both items and key items. If a state is not specified, will alternate.
Example Syntax: /fa config keyitemprefix enabled


Syntax: /fa config writedelay [Required: delay]
Arguments:
delay: The minimum length of time, in milliseconds, FindAll will wait before writing inventory changes to disc when CacheToDisc is enabled. Must be between 0 and 300000.
Effects: Changes the amount of time after an inventory change before it is cached to disc. This is used so that frequent inventory changes do not each require a disc write.
Example Syntax: /fa config writedelay 20000


Syntax: /fa clear
Arguments: N/A
Effects: Clears all searches that are currently displayed in the ImGui element. Does nothing in chatlog displaymode.
Example Syntax: /fa clear


Syntax: /fa search [Required: Search Term] [Optional: additional search term(s)]<
Arguments:
Search Term: An item to search for. You can specify item ID, item name, or a wildcard string that will represent multiple item names.
Effects: Searches all character caches and your local character for instances of the items and displays the results according to the current display mode. Be aware that an extremely broad search(such as /fa search *) may crash or lag the client in chatlog mode due to the volume of data being printed.
Example Syntax: /fa search *crystal


Syntax: /fa local [Required: Search Term] [Optional: additional search term(s)]<
Arguments:
Search Term: An item to search for. You can specify item ID, item name, or a wildcard string that will represent multiple item names.
Effects: Searches your local character for instances of the items and displays the results according to the current display mode. Be aware that an extremely broad search(such as /fa search *) may crash or lag the client in chatlog mode due to the volume of data being printed.
Example Syntax: /fa local *crystal
